功能介绍

功能：

简要介绍此功能的用途和意义。与编写用户手册一致。

操作：

详细描述功能的具体操作步骤，每一个操作步骤应该简单明了，用户根据操作步骤能学会使用该功能，并能体会到该功能的用途和意义。

格式为：

  1. 功能的启动；
  2. 功能的使用（步骤若干）；
  3. 功能的退出。

说明：

需要说明和注意的内容（使用项目符号或者列表编号均可）。一般将与该功能介绍内容有关的备注、说明、注意或者强调等信息记录于此。

在编写系统功能介绍时应遵循以下原则：

  1. 应该以系统的功能为主线进行说明介绍；
  2. 应该先对每一个模块功能进行简单介绍，然后再介绍模块中的具体功能；
  3. 功能介绍的顺序应依据功能实现的逻辑顺序进行，不应单纯地按照菜单项、按钮或工具条的顺序介绍；
  4. 应该详细地描述操作过程中的每一步。
