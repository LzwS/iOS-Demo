帮助模板

注意事项

帮助

  1. 联机帮助是应用程序中不可缺少的一部分，一个友好的联机帮助文档应该形象、直观、快捷、方便地提供给用户所需的帮助信息；
  2. 在开始编写帮助文件之前，需要设计帮助文件的框架和结构。帮助主题规划是整个帮助制作过程中最重要的一步，就如同系统设计、用户手册的结构编排等；
  3. 确保用户通过帮助目录和主题中的链接，或者通过搜索，能找到与系统相关帮助信息；
  4. 帮助目录结构合理，一般的帮助目录结构级次多为3级，可视系统的复杂度增删至4级或者2级，但是建议目录结构级次最好不要超过3级；
  5. 主题设计尽量实用，便于用户查找，尽量一个窗口对应一个独立主题，在对应窗口下可直接调出相关主题；
  6. 主题的别名和编号要有意义和规律，便于维护和管理，参见编号规范；
  7. 尽可能全面地提供索引关键字，加快搜索速度；
  8. 联机帮助应该有一个合适的首页（封面），首页一般只有一页内容，不要有横竖滚动条；
  9. 联机帮助中一般不插入图片，基本为文字介绍。

主题文件

  1. 主题文件的命名最好使用英文，防止帮助制作工具不能辨别，并且命名要有意义，便于查找和维护；
  2. 在编写帮助文件的时候，所有主题都应该采用同一个CSS样式表单，来保证帮助风格的一致；
  3. 主题文件中，能提供链接的地方尽量多提供，方便用户直接查找；
  4. 对于主题文件中有可能链接系统以外网页的，一般使用新窗口打开该网页，而非直接在chm窗口中打开，如：本系统中www.jiuqi.com.cn为引用系统外网页；
  5. 主题文件中无序编号后一般没有标点，有序编号后用“；”隔开，最后一个编号用“。”结束；
  6. 每一个主题文件中的内容不要过多，最多为chm主题区域大小的3页内容；
  7. 插入到主题文件中的图片不能进行缩放，否则图片中的字体会变形。

CSS

  1. 所有公用的样式应该都写到一个CSS文件中，CSS样式选择符的命名应该简单明了；
  2. 如果希望在html文件中使用已经定只好的CSS样式表单时，一定要记住在html文件的“<head></head>”部分加入CSS文件的引用哦；
  3. 在html文件中，使用id=”样式名称”来引用CSS中用“#”定义的样式，使用class=”样式名称”来引用CSS中用“.”定义的样式，样式名称前带有“#”的优先级高于带有“.”的优先级；
  4. TD元素的noWrap属性的行为与TD元素的width属性有关：如果未设置TD宽度，则noWrap属性是起作用的，如果设置了TD宽度，则noWrap属性是不起作用的。

其他

  1. 公用的脚本函数应该写到.js文件中，只需在使用的时候调用.js文件即可；
  2. 当你打开已经编译好的.chm文件，突然发现有图片不能显示，这是什么原因呢？
    首先，可能是你在html中引用图片的时候，路径没有写正确，修改为正确路径即可，最好写绝对路径哦；其次，可能该图片是在CSS样式表单或者.js文件中引用的，而非在html文件中直接引用，这时你必须在任意一个html文件中引用一下该图片，但是让其不显示即可。
